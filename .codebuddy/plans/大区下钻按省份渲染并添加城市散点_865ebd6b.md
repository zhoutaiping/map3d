## 产品概述

优化大区下钻功能，点击大区后只显示该大区内的省份轮廓，并为每个大区添加 4 个随机城市散点。

## 核心功能

1. **省份级大区渲染**：下钻到大区时，仅绘制该大区包含的省份边界（省级），不加载市级区域地图数据
2. **随机城市散点**：从该大区的省份中随机选取 4 个省份，在每个省份中心点位置显示城市散点
3. **统一散点样式**：散点使用统一颜色和大小，悬停时显示省份名称提示

## 技术栈

- 框架：Vue 3 + TypeScript
- 图表库：ECharts + echarts-gl (3D地图)
- 地图数据：GeoJSON (china.json)

## 实现方案

### 核心改动

修改 `renderRegionGroupDrillDown` 函数的实现逻辑：

- **当前逻辑**：加载该大区所有省份的市级区域地图数据（fetch `/maps/${adcode}.json`），合并成包含市级区域的 GeoJSON
- **新逻辑**：直接从 `china.json` 中筛选该大区的省份边界数据，不加载市级地图

### 关键技术点

1. **省份边界数据提取**

- 从已导入的 `chinaJson` 中筛选 `regionGroup.provinces` 对应的 features
- 使用这些 features 创建新的 GeoJSON 注册为地图

2. **省份中心点计算**

- 复用现有的 `getFeatureCenter` 函数计算每个省份的中心点
- 将中心点作为城市散点的位置

3. **随机城市选取**

- 从该大区的省份数组中随机打乱并取前 4 个
- 使用 Fisher-Yates 洗牌算法确保随机性

4. **散点样式统一**

- 固定颜色：使用大区对应颜色或统一标识色
- 固定大小：`symbolSize` 设为常量值
- Tooltip：显示省份名称

### 性能优化

- 不再加载市级地图数据，减少网络请求
- 直接使用内存中的 `chinaJson` 数据，提升响应速度
- 散点数量固定为 4 个，避免过多渲染开销
